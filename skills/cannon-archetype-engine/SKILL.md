# Cannon Archetype Engine

Use this skill whenever a user asks to build, repair, convert or explain a named modern cannon family.

## First command

```powershell
python scripts/cannon-archetype-engine.py audit --archetype <archetype-id>
```

Do not start geometry edits until the capability audit and exact reference hash are known.

## Workflow

1. Resolve the archetype in `knowledge/archetypes/registry.json`.
2. Verify the exact private reference SHA-256.
3. Read its evidence ceiling and forbidden shortcuts.
4. Generate a diagnostic plan with `cannon-archetype-engine.py plan`.
5. Preserve the source and map modules before changing anything.
6. Apply exact mode controls after empty settling and before fill.
7. Fire through the native input.
8. Verify the registered dispenser-cohort fingerprint.
9. Measure the family-specific output contract.
10. Change one causal variable at a time.
11. Promote only when every required capability and runtime contract passes.

## Rev-Worm rules

For `rev-worm-383-v4`:

- preserve north/south opposing panels;
- preserve observer and slime-motion order;
- require master lever, both REV/NUKE mode levers and native button activation;
- verify the 537, 336 and 144 dispenser cohorts at relative offsets 0, +2 and +3 ticks;
- do not require falling blocks for the tested REV/NUKE mode;
- do require a stable TNT output corridor before calling it a worm;
- do not mix Asser east/west modules without interface proof;
- do not delete dispensers or stretch one panel to satisfy EC160;
- never call a cohort-only PASS a working output.

## Fail-closed language

Use these labels precisely:

- `static-reference`: decoded geometry only;
- `local-runtime`: reproduced on the exact public Paper/Sakura setup only;
- `diagnostic-prototype`: experiment candidate, no functionality claim;
- `local-candidate`: all local family contracts pass repeatedly;
- `ec-ready`: only after a passing live ExtremeCraft canary.

The current Rev-Worm contract is capped at `diagnostic-prototype` because clean directionally stable output has not yet been reproduced.
